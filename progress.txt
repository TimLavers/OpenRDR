# Cucumber-suite stabilisation — handoff note (2026-04-30)

## Session goal
Fix regressions / flakes surfaced by `./gradlew cuT` after earlier work to
replace accessibility-tree-based cucumber probing with Compose test hooks
for performance.

## What was changed in this session (all committed to working tree,
## unpushed unless noted)

1. `cucumber/src/test/kotlin/io/rippledown/integration/pageobjects/CornerstonePO.kt`
   - `requireCornerstoneLabel` now formats exactly like
     `CornerstoneInspection`: `"Cornerstone ${index+1} of $total"` when
     `total > 0`, bare `"Cornerstone"` when `total == 0`.
   - Root cause of the original regression: when porting to the
     `CornerstoneTestHook`, the label was computed as
     `"Cornerstone $index of $total"` (0-based). The UI shows 1-based,
     so every cornerstone scenario that asserted the label string failed.

2. `ui/src/main/kotlin/io/rippledown/chat/ChatTestHook.kt`
   - Removed two stray imports the IDE had auto-added
     (`import io.rippledown.chat.ChatTestHook.snapshotRef`,
     `... .update`). `snapshotRef` is `private`; the imports compiled as
     no-ops but were misleading.

3. `server/src/main/kotlin/io/rippledown/kb/chat/ChatManager.kt`
   - `startConversation` now mirrors `processConversationResponse`:
     calls `extractJsonFragments(response)` first, and if there are no
     JSON fragments (or parsing fails), surfaces the raw text as
     `ChatResponse(response)` instead of throwing.
   - Root cause of failure this fixes: the LLM returns **prose** (not
     JSON) from `startConversation` when the case already has comments
     (message starts "This case has the following comments: ...").
     Previous code fed that prose directly to
     `fromJsonString<ActionComment>()`, which threw
     `JsonDecodingException`, which bubbled up as HTTP 500 on
     `/api/startConversation`, which caused the client to wedge.
     Server logs confirmed this — see the
     `/api/startConversation` error stacktrace captured in a prior
     `cuT` run.

4. `server/src/test/kotlin/io/rippledown/kb/chat/ChatManagerTest.kt`
   - Added two regression tests for the above:
     - `startConversation should surface a prose response as a plain
       bot message rather than 500` — exact-match for the failure mode.
     - `startConversation should surface malformed JSON-looking text
       rather than 500` — defensive against the same class of bug.

5. `server/src/main/resources/chat/instructions/3_defining_the_report_change.md`
   - Strengthened the quoted-comment rule (Step 3) with a new
     language-agnostic clause naming the exact Spanish and French
     phrases from the cucumber scenarios. MUST emit `ADD_COMMENT`,
     MUST NOT call `TRANSFORM_REASON` on the quoted text.

6. `server/src/main/resources/chat/instructions/examples/non_english_comment.md` (NEW)
   - Full Spanish few-shot example showing:
     * `User: Add the comment: "La paciente presenta diabetes gestacional."`
       → `ADD_COMMENT` verbatim (the exact failing scenario input,
       now demonstrated as correct in-context).
     * Spanish free-text reason routed through `TRANSFORM_REASON`
       (reinforces the distinction, not just the prohibition).

7. `server/src/main/kotlin/io/rippledown/kb/chat/KBChatService.kt`
   - Added `"non_english_comment.md"` to `systemPromptExampleSections`
     so (6) actually ships in the system prompt.

## Pre-existing behaviour retained (do NOT weaken)
- `CornerstoneInspection`'s 1-based cornerstone label formatting is
  the source-of-truth; `CornerstonePO.requireCornerstoneLabel` must
  stay in sync with it.
- The `@delay_after_cuke` tag is still applied per-scenario in the
  chat feature files to avoid Gemini rate-limiting; this adds real
  wall-clock time but is deliberate.
- The `CornerstoneTestHook` / `ChatTestHook` pattern is the new
  preferred way to observe Compose state from cucumber — do not
  revert POs to accessibility-tree polling.
